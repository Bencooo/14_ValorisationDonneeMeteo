# Politique de sécurité

## Versions prises en charge

Nous corrigeons les vulnérabilités de sécurité sur la branche **`main`** (dernière version en développement).

## Signaler une vulnérabilité

Ne pas ouvrir d’issue publique GitHub pour un problème de sécurité sensible.

1. Utiliser l’onglet **Security** du dépôt GitHub → **Report a vulnerability** (advisory privée), ou  
2. Contacter l’équipe **Data For Good** via les canaux du projet (Slack / site [dataforgood.fr](https://dataforgood.fr)).

Indiquez si possible : description du problème, étapes de reproduction, impact estimé.

Nous nous efforçons de répondre sous **quelques jours ouvrés**.
